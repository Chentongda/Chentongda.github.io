ArchLinux 上使用 Clash 订阅

ArchLinux 上使用代理，方便使用 Github、Google。

CLI 上使用代理参考 ssr-command-client

1 Archlinux 安装 Clash

1.1 AUR 搜索到 Clash，执行安装，这样可以自动生成 service 文件，方便通过 systemctl 管理 clash

yaourt -Ss clash yaourt -S clash

操作演示

1.2 启动

在配置好 service 文件后，systemctl 控制命令：

  • sudo systemctl enable clash@<username> 开机自启
  • sudo systemctl start clash@<username> 启动
  • sudo systemctl stop clash@<username> 停止

在默认情况下，Clash 会监听三个端口：

  • http 代理端口：7890

  • socks 代理端口：7891

  • clash restful 配置服务端口：9090

    此端口一般是提供给Dashboard使用

1.3 配置文件

配置文件的路径： ~/.config/clash/config.yaml
下面生成的配置信息要拷贝到这个配置文件中

2 Clash 订阅

2.1 Clash 订阅导入原理

在 Linux 中导入订阅需要用到两个工具：

  • subconverter
    subconverter 的作用是将订阅连接转换成 Clash 配置
  • sub-web
    sub-web 是 subconverter 的前端，提供了一个方便的浏览器页面

2.2 本地安装 sub-web

sub-web 是一个nodejs 项目，为了编译这个项目，需要安装 nodejs 环境并安装 yarn

yaourt -Ss nodejs-lts yaourt -S nodejs-lts-gallium

操作演示

从 github clone 这个项目后，使用 yarn install yarn serve 就能启动

  • yarn 用 npm 安装；先预装 npm
    yaourt -S npm
操作演示
  • 使用 npm 全局安装 yarn
    npm install --global yarn
操作演示

  • 安装 sub-web

    git clone https://github.com/MSKNET/sub-web.git
    yarn install yarn serve

操作演示

2.3 安装 subconverter

作者提供了 release，直接下载 subconverter_linux64.tar.gz

tar xvfz subconverter_linux64.tar.gz -C ~/ 文件解压后在目录下执行 subconverter 命令，就可以启动服务，缺省端口 25500

操作演示

3 使用

按照上述步骤启动 sub-web 和 subconverter 之后，访问 sub-web 的浏览器地址，这个地址在执行 yarn serve 后会在终端提供，默认是 http://localhost:8080/

可后台执行命令

将生成的“定制订阅”链接拷贝到新标签页，可以得到完整的 Clash 配置

将其拷贝到 ~/.config/clash/config.yaml 文件中，重启 Clash 即可使用
浏览器使用 SwitchyOmega 进行代理

4 通过 clash-dashboard 管理代理

订阅链接中一般提供很多的代理服务器，Clash 会自动选择最快的一个，想要自主选择使用某台代理服务器，需要用到 clash-dashboard

和 sub-web 一样，这也是一个前端项目，依次执行 yarn install yarn start 就可以了
由于这个项目使用的是 vite 构建，可能需要执行 yarn global add vite 安装
git clone https://github.com/Dreamacro/clash-dashboard.git
yarn install yarn start

操作演示

5 使用 Proxychains

配置文件： /etc/proxychains.conf
yaourt -Ss proxychains yaourt -S proxychains
将本地代理写入 proxychains 配置文件中 sudo vim /etc/proxychains.conf
使用格式： proxychains curl www.google.com
